# Build sequence philosophy: hybrid (build for Nightcap, design platform-clean from day one)

Date: May 7, 2026
Rationale: Build only what Nightcap needs at MVP, but design every interface as platform-clean from line one. Avoids over-engineering a full platform pre-proof while preventing the 'extract platform later' anti-pattern. Forces discipline on layer separation. Alternatives considered: platform-first (too speculative for solo founder), Nightcap-only with later extraction (creates technical debt that becomes rewrites).
Section: Cross-cutting
Status: Committed
