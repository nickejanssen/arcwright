# Session interruption handling: partial persistence (character and knowledge state saved, arc restarts from nearest designed beat)

Date: May 5, 2026
Rationale: On interruption, platform saves character state and knowledge state. On resume, arc restarts from the nearest designed beat. Sessions never restart from the beginning; no state is permanently lost. Full mid-scene resume is deferred as scope debt; beat-level resume covers the most common interruption scenarios and is sufficient for MVP.
