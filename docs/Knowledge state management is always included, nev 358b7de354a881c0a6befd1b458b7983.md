# Knowledge state management is always included, never optional; schema complexity is what is tiered by plan

Date: May 5, 2026
Rationale: Making knowledge state optional would allow developers to ship experiences where AI characters reveal information they should not know, producing incoherent sessions that damage the platform reputation. The platform always enforces knowledge boundaries; the developer defines the schema. Complexity of schema determines cost tier. Competitor failures at this problem (Inworld, Convai) are the evidence base.
