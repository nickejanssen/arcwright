# Unified character model with split behavior source

Date: May 5, 2026
Rationale: Every character (human-controlled or AI-driven) is the same object type: identity, personality profile, goals, knowledge state, relationship graph. Behavior source differs: AI characters have a generation engine; human characters have an input channel. Required for the monster RPG (player and NPC are interchangeable) and for Nightcap (players are characters in the story, not observers).
