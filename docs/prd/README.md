# PRD Directory

This directory contains the Product Requirements Document split into focused sections for easier navigation and versioning.

## Files

- **01-overview.md**: Product vision and three-horizon roadmap
- **02-requirements.md**: Architecture principles, Nightcap reference implementation, platform capabilities
- **03-scope.md**: MVP definition, milestone map, Visual Storyworld roadmap, business model
- **04-non-goals.md**: Architectural constraints and deferred decisions

Related product logs live in [../product/](../product/). Raw full-document exports live in [../archive/notion-export/](../archive/notion-export/).

## Adding New PRD Files

- Follow the naming convention: `NN-description.md` (e.g., `05-pricing-model.md`)
- Each file should have a metadata block at the top:
  ```
  > Source: Project knowledge artifact 06-PRD-v1.3
  > Last synced: YYYY-MM-DD
  > Do not edit this file directly; edit in Notion and re-sync.
  ```
- Use H1 headings for main sections
- Link between related files using relative markdown links
- Keep metadata synced when upstream PRD is updated
