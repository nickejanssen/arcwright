# Games are deliberate platform stress-tests; each game's design brief must explicitly name which platform capabilities it validates

Date: May 5, 2026
Rationale: The platform roadmap is sequenced around validation targets, not just creative preference. Nightcap validates linear arc execution, per-player knowledge state, multi-character ensemble, multi-surface routing, dynamic pacing. Monster RPG validates world state persistence, emergent narrative, procedural generation, player-defined motivation inference. Third game selection evaluated against remaining capability gaps. This discipline distinguishes platform-by-design from platform-by-accident.
